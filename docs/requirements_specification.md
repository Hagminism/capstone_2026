### 기능적 요구사항

1. **사용자 인증**
    - 사용자는 이메일을 통한 회원가입과 로그인 또는 소셜 로그인(구글, 네이버, 애플, 카카오톡 등)이 가능해야한다.
    - 회원가입 시에는 본인이 일반 사용자인지, 혹은 관리자인지를 선택하여 회원가입 할 수 있어야 한다.
        - **(공통)** 사용자는 이메일, 비밀번호, 이름, 휴대폰 번호, 필수 약관 동의 내역 등의 기본 정보를 입력한다.
        - 관리자로 회원가입을 진행할 경우 본인이 운영하고 있는 업장에 대한 인증을 진행해야한다.
            - 사용자(관리자)는 상호명, 업장 종류, 사업자등록번호 10자리, 대표자명, 개업일자, 업장 주소 및 연락처, 운영 시간 등의 필수 정보를 추가로 입력한다.
            - 국세청에서 제공하는 사업자등록정보 진위확인 및 사업자등록 상태조회 API 서비스를 통해 현재 실제로 업장을 운영 중인지를 검증한다.
            - **(추가)** 사용자(관리자)는 사업자등록증 사진을 첨부하고, 플랫폼 관리자가 정보 대조 후 문제가 없으면 승인하는 방식으로 추가 검증을 진행한다.
    - **(추가)** 비회원 사용자에게 둘러보기를 허용할 경우, 인증 정보가 필요한 기능에 접근 시 인증을 요구해야 한다.
    - 주류를 판매하는 업장, 특정 시간대 이후 미성년자의 출입이 제한되는 업장 등의 경우, 사용자가 해당 업장의 예약 페이지에 진입하거나 예약을 시도하려는 시점에서 인증을 요구해야 한다.
2. **홈 화면**
    - 상단에 검색 바를 배치하여 위치에 상시 업장 검색이 가능해야 한다.
        - 검색 바를 탭 할 시 검색 페이지로 이동한다.
    - 카테고리 별 업장 확인이 가능해야 한다.
    - 우측 상단 알림 아이콘을 통해 최근 수신한 알림을 확인할 수 있어야 한다.
        - 아이콘을 탭 할 시 알림 페이지로 이동하며 목록 형태로 최근 알림을 표시한다.
        - 확인하지 않은 알림은 유채색으로, 확인한 알림은 무채색으로 표시한다.
        - **(추가)** 타 기능과 연관이 있는 알림일 경우 해당 페이지로 이동시킨다.
3. **검색 페이지**
    - 상단에 검색 바를 배치하여 위치에 상시 업장 검색이 가능해야 한다.
        - 검색 바를 계속 표시하고, 사용자가 입력하는 텍스트를 그대로 텍스트 필드에 반영해야 한다.
        - 사용자가 입력한 텍스트와 일치하는(혹은 연관된) 업장을 검색 바 하단에 목록으로 표시해야 한다.
            - 항목 당 해당 업장의 업체명, 사진, 주소를 함께 표시해야한다.
            - 특정 업장 항목을 탭 할 시 해당 업장의 정보 페이지로 이동해야 한다.
4. **업장 정보 페이지(혹은 예약 페이지)**
    - 페이지 최초 로드 시에 업장과 관련된 정보들이 표시되어야 한다.
        - 상단에는 관련 사진들을 가로로 스크롤하며 확인 할 수 있어야 한다.
        - 중단에는 업체명, 업종(+식당이라면 음식 종류), 별점, 리뷰 개수, 운영 시간 등이 표시되어야 한다.
        - 하단 네비게이션 바에는 저장(북마크) 버튼, 예약하기 버튼이 표시되어야 한다.
    - 수직으로 스크롤해서 화면을 내리면 상단에 가로 메뉴 바를 통해 업장 관련 정보들을 제공해야 한다.
        - 홈, 메뉴, 사진, 리뷰, 매장 정보 등을 가로 메뉴 바에 배치한다.
        - 홈 탭에는 나머지 탭(메뉴, 사진, 리뷰, 매장 정보 등)의 정보를 수직으로 간략하게 나열한다.
        - 메뉴, 사진, 리뷰 탭에서는 각 항목들을 목록으로 표시한다.
        - 매장 정보 탭에는 관리자가 관리자 페이지에서 작성한 내용을 그대로 표시한다.
    - 예약하기 버튼을 누르면 바텀 시트가 올라오면서 날짜, 시간, 인원 등을 선택할 수 있어야 한다.
5. **내 주변 화면(지도 페이지)**
    - 페이지 최초 로드 시에는 사용자의 현 위치를 기준으로 정보를 제공한다.
        - 사용자의 현 위치가 화면의 중심에 오도록 표시한다.
        - 사용자의 현 위치를 기준으로 기본 반경 값(ex: 800m, 1km 등) 내의 등록 업장들을 화면에 핀으로 표시한다.
    - “이 지역에서 (재)검색” 버튼을 배치하여, 지도를 스크롤하여 이동했을 때 해당 지역을 기준으로 업장을 재검색 할 수 있어야 한다.
        - 화면의 정중앙을 기준으로 기본 반경 값(ex: 800m, 1km 등) 내의 등록 업장들을 화면에 핀으로 표시한다.
    - 특정 기준에 따라 화면에 표시되는 업장을 필터링하여 표시할 수 있어야 한다.
        - 업장 종류(식당, 카페, 스터디 카페)에 따른 필터링이 가능해야 한다.
        - 별점(rating)에 따른 필터링(ex: 별점이 4.0 이상인 업장만 표시)이 가능해야 한다.
        - 식당의 경우 해당 업장에서 제공하는 음식의 종류(ex: 한식, 중식, 일식, 양식, 베트남 음식 등)에 따른 필터링이 가능해야 한다.
        - 운영 시간에 따른 필터링(ex: 현재 운영 중인 업장만 표시)이 가능해야 한다.
    - 검색 반경 수치 값을 변경할 수 있어야 한다.
        - 값을 변경할 경우 즉시 재검색을 실시하여, 사용자의 현 위치를 기준으로 반경 값 내의 등록 업장들을 화면에 핀으로 표시한다.
    - 상단에 검색 바를 배치하여 위치에 상시 업장 검색이 가능해야 한다.
        - 검색 바를 탭 할 시 검색 페이지로 이동한다.
6. **저장(찜) 페이지**
    - 내가 저장한 업장을 목록 형태로 조회할 수 있어야 한다.
        - 해당 업장의 업체명, 사진, 카테고리, 별점 및 별점 개수 등을 간단히 조회할 수 있어야 한다.
    - 저장 버튼(북마크)을 다시 눌러 비활성화 할 경우 해당 업장이 목록에서 제거되어야 한다.
7. **마이페이지(혹은 설정 페이지)**
    - 사용자의 기본 정보(프로필 정보)를 수정할 수 있어야 한다.
        - 수정 버튼을 누르면 기본 정보 수정 페이지로 이동하여 수정을 진행한다.
    - 사용자의 이용 내역 및 리뷰 내역을 확인할 수 있어야 한다.
        - 이용 내역 및 리뷰 내역 조회 버튼을 누르면 조회 페이지로 이동하여 목록 형태로 내역을 표시한다.
    - 알림 설정, 1:1 문의, 공지사항 및 이용약관 조회 등이 가능해야한다.
        - 알림 설정 페이지로 이동하여 스위치 토글 방식 등을 통해 알림 설정 범위를 수정할 수 있어야 한다.
        - 1:1 문의는 이메일 문의 방식으로 진행하며, 플랫폼 관리자가 내용 확인 후 조치를 진행한다.
        - 공지사항은 전용 페이지에서 목록 형태로 표시하며, 항목을 탭 할 경우 확장하는 형태로 상세 내용을 표시한다.
        - 이용약관은 외부 페이지로 이동시키거나 웹뷰를 통해 표시한다.
    - 우측 상단 알림 아이콘을 통해 최근 수신한 알림을 확인할 수 있어야 한다.
        - 아이콘을 탭 할 시 알림 페이지로 이동하며 목록 형태로 최근 알림을 표시한다.
        - 확인하지 않은 알림은 유채색으로, 확인한 알림은 무채색으로 표시한다.
        - **(추가)** 타 기능과 연관이 있는 알림일 경우 해당 페이지로 이동시킨다.
    - **(추가)** 사용자의 결제 수단 정보를 수정할 수 있어야 한다.
